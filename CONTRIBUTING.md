# Contributing to Penko-tune

Thank you for your interest in contributing! Penko-tune is built by the community, for the community.

---

## Ways to Contribute

### 1.  Report Bugs
- Check [existing issues](https://github.com/yourusername/penko-tune/issues) first
- Provide reproduction steps
- Include browser/OS info
- Screenshots help!

### 2.  Suggest Features
- Open an issue with the "enhancement" label
- Explain the use case
- Check [UPGRADE_PROPOSAL.md](UPGRADE_PROPOSAL.md) first

### 3.  Submit Code
- Fork the repo
- Create a feature branch: `git checkout -b feature/amazing-feature`
- Make your changes
- Test thoroughly
- Submit a pull request

### 4.  Improve Documentation
- Fix typos
- Add examples
- Improve clarity
- Translate to other languages

### 5.  Design & UI
- Submit theme designs
- Create visualizer effects
- Design promotional graphics
- Improve UX

---

## Development Setup

```bash
# Clone your fork
git clone https://github.com/yourusername/penko-tune.git
cd penko-tune

# Install dependencies
npm install

# Start dev server
npm run dev

# Build for production
npm run build
```

---

## Code Guidelines

### General Principles
- **Privacy first** - No tracking, no data collection
- **Keep it simple** - Avoid over-engineering
- **Performance matters** - Optimize bundle size
- **Accessible** - Follow WCAG guidelines

### Code Style
- Use TypeScript for type safety
- Follow existing formatting (React functional components)
- Comment complex logic
- Use descriptive variable names

### Commit Messages
```
feat: Add Lightning payment integration
fix: Resolve equalizer reset bug
docs: Update deployment guide
refactor: Simplify audio context initialization
```

---

## Pull Request Process

1. **Update documentation** if needed
2. **Test your changes** thoroughly
3. **Keep commits atomic** (one feature/fix per commit)
4. **Reference issues** in PR description
5. **Be responsive** to feedback

### PR Template

```markdown
## Description
Brief description of changes

## Type of Change
- [ ] Bug fix
- [ ] New feature
- [ ] Breaking change
- [ ] Documentation update

## Testing
- [ ] Tested on Chrome
- [ ] Tested on Firefox
- [ ] Tested on Safari
- [ ] Tested on mobile

## Checklist
- [ ] Code follows project style
- [ ] No console errors
- [ ] Documentation updated
- [ ] No new dependencies (or justified)
```
